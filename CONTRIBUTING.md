# Contributing to PixelPal

We love your input! We want to make contributing to PixelPal as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features

## Our Development Process

1. Fork the repo and create your branch from `main`.
2. Write clean, self-documenting code.
3. Test your changes locally (running a local web server to check modules and PWA service workers).
4. Issue that Pull Request!

## Code Style

- Use ES Modules with clean imports.
- Maintain CSS variables inside `style.css`.
- Keep component/module responsibilities isolated.
- Avoid introducing external frameworks or heavy libraries.
